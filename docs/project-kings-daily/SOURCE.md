# Source fit and buffer

## Откуда брать

Бери кандидатов только из существующих настроенных source pools и одобренных маршрутов канала. Не ищи новые доступы и не меняй source policy внутри daily run. YouTube-поиск допустим только для THE LIGHT KINGDOM и только если он уже разрешён текущей настройкой.

До старта code-preflight требует минимум шесть готовых источников на каждый канал. Refiller пополняет запас заранее и после `3×3`; нехватка не должна впервые обнаруживаться посреди выпуска.

## Исполнитель source fit

Исполнитель получает чистый пакет: точный channel ID, URL и source ID, метаданные, transcript/caption если есть, репрезентативные кадры, длительность, crop-зоны и недавние `story_event_id`. Он проверяет:

1. точное соответствие `CHANNELS.md`;
2. один понятный event с hook → development → payoff;
3. видимость главного действия после вертикального crop;
4. отсутствие неподтверждаемых ключевых утверждений;
5. отсутствие дубля URL, media hash или сюжета;
6. возможность убрать donor UI, CTA, watermark и чужие captions без потери действия.

Верни только короткий артефакт:

```json
{
  "decision": "PASS|REPLACE",
  "summary": "one factual sentence",
  "visibleEvidence": ["visible fact 1", "visible fact 2"],
  "factualClaims": ["claim supported by the input"]
}
```

Не добавляй скрытые рассуждения и не улучшай слабый источник выдуманным текстом.

## Независимый судья

Судья видит тот же вход, артефакт исполнителя и критерии, но не видит сообщения или рассуждения исполнителя. Он возвращает только `{"verdict":"PASS|REWORK|REPLACE","issues":["точная проблема"],"summary":"краткий итог"}`. После двух `REWORK` источник обязательно заменяется.

## Состояние и замена

- `ready`: источник прошёл исполнителя и судью и не является дублем.
- `reserved`: закреплён за одним slot; другой maker его не берёт.
- `used`: ставится только после `public_verified` на правильном YouTube-канале.
- Ошибка render, schedule, upload или QA не переводит источник в `used`.
- При обычной ошибке скачивания — до трёх коротких повторов; затем `REPLACE`.
- Concept mismatch, неустранимый donor overlay, отсутствие payoff, дубль или два исчерпанных semantic rework сразу ведут к `REPLACE`.
